# IMP-014: CLAUDE.md — réduire section backlog au strict minimum

**Type:** Improvement
**Statut:** A faire

**Complexite:** XS
**Tags:** perf
**Date creation:** 2026-03-21

## Description

La section backlog dans `~/.claude/CLAUDE.md` (~50 lignes, ~350 tokens) contient les tables de priorité, complexité et conventions qui sont AUSSI présentes dans les commandes slash `/backlog-bug`, `/backlog-feat`, `/backlog-imp`. Quand l'utilisateur crée un ticket via la commande slash, ces tables sont chargées deux fois.

Réduire la section CLAUDE.md au workflow + commandes disponibles. Les détails (tables, template) restent dans les commandes slash où ils sont utiles.

## Approche

Remplacer la section `<!-- backlog:start -->` ... `<!-- backlog:end -->` par une version minimale :

```markdown
<!-- backlog:start -->
## Systeme de ticketing (BACKLOG)

BACKLOG/ contient BUGS/, FEATURES/, IMPROVEMENTS/ avec PENDING/ et DONE/.
INDEX.md est un cache — ne JAMAIS l'editer manuellement.
Les IDs sont calcules par scan des fichiers existants (anti-conflit multi-sessions).
Le hook backlog-guard bloque les conflits d'ID.

Commandes : `/backlog-bug`, `/backlog-feat`, `/backlog-imp`, `/backlog-status`, `/backlog-init`
<!-- backlog:end -->
```

~6 lignes au lieu de ~50. L'essentiel (workflow, anti-conflit, commandes) est préservé.

## Fichiers concernes

- Les commandes slash backlog — vérifier qu'elles contiennent bien les tables détaillées
