# IMP-027: install.sh — ajouter les 4 modules manquants au MODULES array

**Type:** Improvement
**Statut:** A faire
**Priorite:** Haute
**Complexite:** XS
**Tags:** installer, modules
**Date creation:** 2026-03-22

---

## Description

Le `MODULES` array dans `install.sh` (ligne 34) ne contient que 8 modules. 4 modules avec un `install.sh` fonctionnel ne sont pas listés et ne sont donc pas installés par `bash install.sh --all` :

- `setup-project`
- `api-contract`
- `scope-enforcer`
- `post-tool-use`

Conséquence : un utilisateur qui fait `install.sh --all` n'obtient pas une installation complète.

## Approche

1. Ajouter les 4 modules au `MODULES` array avec leurs `DESCRIPTIONS` et `DEPS` correspondants
2. Vérifier que chaque `install.sh` de ces 4 modules fonctionne en standalone (déjà testé, mais vérifier)
3. L'ordre dans le array n'a pas d'importance (pas de dépendances inter-modules)

## Fichiers concernes

- `install.sh` — arrays MODULES, DESCRIPTIONS, DEPS (lignes 34-54)

## Criteres d'acceptation

- [ ] `bash install.sh --all` installe les 12 modules (8 existants + 4 ajoutés)
- [ ] Chaque nouveau module s'installe sans erreur
- [ ] Le mode interactif affiche les 12 modules avec descriptions correctes
