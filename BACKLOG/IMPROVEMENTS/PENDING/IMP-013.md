# IMP-013: command-guard — appendFile au lieu de read+write pour le log

**Type:** Improvement
**Statut:** A faire
**Priorite:** Basse
**Complexite:** XS
**Tags:** perf
**Date creation:** 2026-03-21

---

## Description

Dans `command-guard/src/cli.ts` lignes 27-37, chaque événement de sécurité :
1. Lit le fichier log entier en mémoire (`await file.text()`)
2. Concatène la nouvelle ligne
3. Réécrit tout le fichier (`await Bun.write(LOG_FILE, existingContent + logLine)`)

C'est O(n) par événement. Après 500 commandes, le fichier grandit et les I/O augmentent. Sur les sessions longues, ça devient mesurable.

## Approche

Remplacer le pattern read+concat+write par un append :

```typescript
// AVANT
const exists = await file.exists();
if (exists) {
    const existingContent = await file.text();
    await Bun.write(LOG_FILE, existingContent + logLine);
} else {
    await Bun.write(LOG_FILE, logLine);
}

// APRES
const fs = await import("node:fs/promises");
await fs.appendFile(LOG_FILE, logLine);
```

`appendFile` crée le fichier si inexistant ET ajoute en fin sans relire. O(1) par événement.
