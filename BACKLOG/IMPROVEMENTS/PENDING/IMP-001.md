## Description

Le module `critical-thinking` fonctionne mais n'est pas encore integre dans l'ecosysteme CLI de claude-conf. De plus, `install.sh` utilise `sed -i ''` qui est macOS-only.

## Taches

### 1. Portabilite sed (Linux)

`critical-thinking/install.sh` ligne 69 utilise `sed -i ''` (syntaxe macOS). Sur Linux, il faut `sed -i` sans argument vide.

**Approche recommandee** : detecter l'OS et adapter la commande sed. Pattern :
```bash
if [[ "$OSTYPE" == "darwin"* ]]; then
    sed -i '' 's/...' "$file"
else
    sed -i 's/...' "$file"
fi
```

Verifier si les autres modules (handoff-kit) ont le meme probleme et uniformiser.

### 2. Root install.sh — ajouter critical-thinking

Ajouter `critical-thinking` dans le tableau `MODULES` et `DESCRIPTIONS` de `install.sh` (root).

### 3. CLI bin/claude-conf — ajouter sync

Ajouter le case `critical-thinking` dans `cmd_sync()` de `bin/claude-conf` pour tracker :
- `claude-md/critical-thinking.md` → `~/.claude/CLAUDE.md` (attention : c'est un append, pas une copie directe — le sync devra comparer le contenu entre markers)

### 4. CLI bin/claude-conf — ajouter disable/enable

Verifier que `disable`/`enable` fonctionne avec critical-thinking (devrait marcher car le CLI detecte les modules par la presence de `install.sh`).
