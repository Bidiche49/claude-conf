# BUG-008: handoff — PreCompact hook génère du JSON invalide

**Type:** Bug
**Statut:** Fait (faux positif — JSON valide, aucune modification necessaire)
**Priorite:** Moyenne
**Complexite:** XS
**Tags:** cli
**Date creation:** 2026-03-21

---

## Description

Dans `handoff-kit/hooks/pre-compact-handoff.sh`, le hook output JSON est construit avec un heredoc shell. Les `\n` dans le heredoc sont des retours à la ligne littéraux, pas des escapes JSON. Le JSON résultant contient des sauts de ligne dans les valeurs string, ce qui est invalide. Claude Code ignore silencieusement le hook output.

## Fichiers concernes

- `handoff-kit/hooks/pre-compact-handoff.sh` — construction du JSON output

## Approche

Option A (recommandée) : utiliser `printf` avec escapes explicites :
```bash
printf '{"result":"backup sauvegarde dans %s"}' "$backup_file"
```

Option B : utiliser `jq` pour construire le JSON (escape automatique) :
```bash
jq -n --arg msg "backup sauvegardé dans $backup_file" '{"result": $msg}'
```

## Criteres d'acceptation

- [ ] Le hook output est du JSON valide (vérifiable avec `jq .`)
- [ ] Le message contient le path du backup

## Tests de validation

- [ ] Déclencher manuellement le hook et piper la sortie dans `jq .`
