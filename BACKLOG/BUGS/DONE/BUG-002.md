# BUG-002: Tab-titles — titre fenetre ecrase par precmd + Terminal.app

**Type:** Bug
**Statut:** Fait
**Priorite:** Haute
**Complexite:** XS
**Scope:** both
**Tags:** tab-titles, hooks
**Date creation:** 2026-03-20

---

## Description

Le titre de la fenetre Terminal.app ne montre pas le dot couleur du projet. Deux causes combinees :

1. Le `precmd` installe par le module (zshrc ligne 241) utilise OSC 0 qui ecrase le titre fenetre et le titre onglet. Chaque fois que le prompt zsh s'affiche, le titre fenetre perd le dot couleur.
2. Terminal.app peut avoir `ShowRepresentedURLInTitle = 1` sur certains profils, ce qui superpose le dossier de travail par-dessus le titre custom.

## Cause racine

### precmd (code bug)

Dans `tab-titles/install.sh`, le bloc ALIASES inclut :
```bash
precmd() {
    printf "\033]0;%s — zsh\007" "$(basename "$PWD")"
}
```

OSC 0 set le titre fenetre ET le titre onglet. Ca ecrase le titre fenetre defini par `_cc_set_win` (OSC 2) des que le shell reprend la main.

### Terminal.app (config user)

`ShowRepresentedURLInTitle = 1` sur un profil ajoute le 📁 + path dans le titre fenetre par-dessus tout.

## Fix

### 1. Corriger precmd dans install.sh

Remplacer le `precmd` actuel par une version qui :
- Utilise OSC 1 (tab seulement) pour le titre onglet quand on est dans un shell normal (pas Claude)
- Ne touche PAS au titre fenetre (OSC 2) — il est deja set par `cc`/`ccs`/`ccd`/`ccw` au lancement et doit rester stable

```bash
precmd() {
    printf "\033]1;%s\007" "$(basename "$PWD") — zsh"
}
```

Ca remet un titre d'onglet lisible quand on quitte Claude (le nom du dossier), sans toucher au titre fenetre.

### 2. Ajouter une note dans install.sh output + README

Rappeler que Terminal.app doit avoir ces reglages :
- Window tab : decocher "Working directory or document"
- Window tab : decocher "Active process name"
