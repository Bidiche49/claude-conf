# BUG-012: Tab-titles — titre d'onglet revient a "CC" au resume de conversation

**Type:** Bug
**Statut:** A faire
**Priorite:** Haute
**Complexite:** M
**Tags:** tab-titles, hooks, session-resume
**Depends on:** none
**Blocked by:** —
**Date creation:** 2026-04-08

---

## Description

Quand une conversation est reprise (`claude --resume`, `/resume`, ou via l'IDE), le titre d'onglet revient a "CC" + couleur projet au lieu de conserver le mode correct (SUP, WORK:TICKET, BIZ).

Le probleme touche TOUS les types de session : supervisor, worker, closing.

## Cause racine

Deux mecanismes de persistance fragiles se combinent :

1. **`session-tab-title.sh` (SessionStart hook)** depend de `$CC_TAB_TITLE` et `$CC_WIN_TITLE`, env vars exportees par les fonctions shell (`ccs`, `ccw`, etc.). Au resume, ces fonctions ne sont pas re-executees → env vars vides → le hook ne fait rien → Claude Code applique son titre par defaut.

2. **`tab-title.sh` (UserPromptSubmit hook)** stocke l'etat dans `/tmp/cc-tab-${PPID}`. Le PPID est le PID du process Claude Code parent. Au resume, c'est un nouveau process → nouveau PPID → nouveau fichier d'etat → default "CC".

Le SessionStart hook recoit un JSON avec `session_id` (stable across resume) et `source` ("resume"|"startup"|"compact"|"clear") mais le jette (`cat > /dev/null`).

## Approche recommandee

### Modification 1 : `tab-title.sh` — persister par session_id

- Extraire `session_id` du JSON stdin (deja disponible, non utilise)
- Remplacer `/tmp/cc-tab-${PPID}` par `.claude-sessions/tab-state/${SESSION_ID}`
- Le state file devient durable et adressable par session_id
- Garder le `/tmp/cc-tab-${PPID}` en fallback lecture seule (compat sessions en cours)

### Modification 2 : `session-tab-title.sh` — restaurer le titre au resume

- Lire le JSON stdin au lieu de `cat > /dev/null`
- Extraire `session_id` et `source`
- Si `source` = "resume" ou "compact" :
  - Lire `.claude-sessions/tab-state/${SESSION_ID}` → restaurer le titre
  - Fallback sur `.claude-sessions/supervisor-active/${SESSION_ID}` → SUP
- Si `source` = "startup" : comportement actuel (env vars CC_TAB_TITLE/CC_WIN_TITLE)
- Utiliser `CLAUDE_ENV_FILE` pour exporter `CLAUDE_CODE_DISABLE_TERMINAL_TITLE=1` au resume

### Alternative ecartee

Persister dans un fichier keye par conversation-id custom : inutile car `session_id` est deja fourni par Claude Code et stable across resume.

## Fichiers concernes

- `tab-titles/hooks/session-tab-title.sh` — ajouter lecture JSON + restauration au resume
- `tab-titles/hooks/tab-title.sh` — remplacer PPID par session_id pour le state file
- `tab-titles/tests/test-tab-titles.sh` — ajouter tests pour le nouveau mecanisme de persistance

## Criteres d'acceptation

- [ ] Au resume d'une session supervisor, le titre affiche "🔴 SUP" (pas "CC")
- [ ] Au resume d'une session worker, le titre affiche "🟢 TICKET-ID" (pas "CC")
- [ ] Au resume d'une session closing, le titre affiche "⚪ BIZ" (pas "CC")
- [ ] Un nouveau lancement via `cc`/`ccs`/`ccw` fonctionne comme avant (pas de regression)
- [ ] Apres un compactage (`source` = "compact"), le titre est preservé
- [ ] Les sessions en cours (avec ancien state file /tmp/) continuent de fonctionner (fallback)
